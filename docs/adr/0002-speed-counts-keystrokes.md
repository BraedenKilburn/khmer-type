# Speed counts keystrokes, not clusters

`សូមស្វាគមន៏!` is 12 keystrokes but only 7 clusters — Khmer routinely needs several keystrokes per visual unit, and COENG needs one while rendering nothing at all. We count keystrokes, because that measures what the fingers actually did and stays comparable to how Latin typing trainers count.

## Consequences

A Khmer line scores meaningfully higher than its visual length suggests, and higher than a Latin line that looks equally long. This reads as inflated and will invite a switch to cluster counting; that would make COENG keystrokes free and undercount real effort. Keep keystrokes.

Any stored speed history is denominated in this unit, so changing it later invalidates comparisons against past sessions.
